Object Oriented Information Systems

Published 30 May 2022

Learning Outcomes

  • Understanding the concept of Data, Information, Knowledge and Information Systems

  • Understanding the fundamentals of Object Oriented Design

  • Understanding how to implement Object Oriented Design in Python

  • Understanding UML design

  • Understanding Database design and SQL

  • Understanding Web development in Python

Reflection

Information System

These units made me understand the true meaning of data and information and how data itself is meaningless without being converted into usable knowledge

It also made me understand the components of an Information System and how the different processes work together.

It also made me understand the true importance of data and how companies intend to "mine" data to produce useful information to further improve their businesses

Object Oriented Design

The module brought a different perspective to the things around me. It made me realize that almost everything is an object in one way or another and they simply just interact which each other.

It also brought the knowledge of how objects can inherit features from a parent object(inheritance) and how some objects can choose to perform differently(polymorphism).

This brought a clearer approach when designing systems or projects by identifying each concrete class or models and determining how they should interact with each other.

It just makes me see things in a clearer way when I implement a system using an Object Oriented approach.

Unified Modelling Language(UML)

UML represents a standard way of representing System Design. It aids the ability to present someone's thought process in a standard way, highlighting all the important components and relationships.

I see it as the best tool to be used for System Design before it highlights all the attributes and relationships between. It is also easy to modify if the need arises.

Database Design and SQL

Database design introduced the concept of ERD (Entity Relationship Diagram) where database designs can be easily represented in a well structured way.

It also introduced the concept of data normalization which enforces some rules to guide database design.

I also learned a lot about SQL databases. How to define relationships and foreign keys and how to properly query and order results

Python Development

This highlighted the Object Oriented features of python including how to implement OOP principles.

This module also explained how to create a real world web application using flask with python.

This gave a more real world view of how a python application will work for a real use case.
